  BUG-001  |  CRITICAL  |  P1  |  FIXED  |  Backend
  TITLE     : H2 v2.4.240 Throws Check Constraint Violation on Simple Inserts
  COMPONENT : Database / H2
  AFFECTS   : BE-002 -> BE-060 (All Repository Tests)
  FOUND BY  : Automated Test Run (ProductRepositoryTest)

  ROOT CAUSE:
     H2 2.4.240 introduced stricter check-constraint validation conflicting with the
     Packora JPA schema. Simple INSERT statements raised SQLState 23514, blocking all
     @DataJpaTest tests.

  STEPS TO REPRODUCE:
     1. Upgrade H2 to 2.4.240 in pom.xml
     2. Run ./mvnw test on any Repository test
     3. Observe: HibernateJdbcException -> Check constraint "ORDERS_STATUS_CHECK" invalid

  FIX APPLIED:
     Pinned H2 to 2.3.232 in backend/pom.xml: <h2.version>2.3.232</h2.version>

  FIXED IN  : backend/pom.xml

  NOTES     :
     Spring Boot 4 auto-resolves to H2 2.4.x; explicit version override is required.

  BUG-002  |  CRITICAL  |  P1  |  FIXED  |  Frontend
  TITLE     : Frontend Tests Fail: TextEncoder/TextDecoder Not Defined in Jest Environment
  COMPONENT : Test Environment / Jest
  AFFECTS   : FE-022 -> FE-048 (All Component Tests)
  FOUND BY  : Automated Test Run (LoginPage.test.jsx)

  ROOT CAUSE:
     React Router v7 requires TextEncoder/TextDecoder Web APIs. Jest's jsdom does not
     include them, causing ReferenceError at import time for any component using routing.

  STEPS TO REPRODUCE:
     1. Create any test using <MemoryRouter> wrapper
     2. Run npm test
     3. Observe: ReferenceError: TextEncoder is not defined

  FIX APPLIED:
     Added polyfills to src/setupTests.js:
     const { TextEncoder, TextDecoder } = require('util');
     Object.assign(global, { TextEncoder, TextDecoder });

  FIXED IN  : src/setupTests.js

  NOTES     :
     Polyfill must be placed before any React Router import in setupTests.js. Order
     matters.
